Project Overview

What Al-Qari is, its purpose, and goals

🕌 What is Al-Qari?

Al-Qari (Arabic: القاري — "The Reciter") is an AI-powered Quranic recitation learning platform that helps Muslims of all levels learn, practice, and improve their Quranic recitation. It combines a Python/FastAPI AI backend with a Flutter mobile app, delivering real-time Tajweed feedback, word-by-word accuracy scoring, and progress tracking.

AI Models Used

Complete breakdown of every AI/ML model powering Al-Qari

ℹ️ Al-Qari uses two primary AI/ML components: a fine-tuned deep learning ASR model for Quranic speech recognition, and an acoustic signal-processing engine for Tajweed rule checking.

🎙️ Model 1 — OdyAsh/faster-whisper-base-ar-quran

Automatic Speech Recognition (ASR) · Hosted on Hugging Face
Active

This is the core AI model of Al-Qari. It is a Faster-Whisper variant (a highly optimized CTranslate2 conversion of OpenAI's Whisper architecture) that has been fine-tuned specifically on Quranic Arabic recitation audio. Standard Whisper models struggle with Quranic Arabic because it uses classical pronunciation, Tajweed articulations, and specific stop/pause patterns not found in conversational Arabic. This fine-tuned model understands those nuances.

What is Whisper?
OpenAI's Whisper is a Transformer-based encoder-decoder neural network trained on 680,000 hours of multilingual audio. The encoder converts raw audio spectrograms into contextual embeddings; the decoder generates token-by-token text output in the target language. The "base" size has ~74M parameters — small enough to run on CPU in real-time while still being highly accurate for constrained Arabic vocabulary (the Quran has ~77,000 words with limited vocabulary).

Why Faster-Whisper?
Faster-Whisper uses CTranslate2, an optimized inference engine that reduces memory usage by 4× and increases speed by 2–4× compared to vanilla Whisper via quantization (INT8 on CPU, FP16 on GPU). For a real-time recitation app, this speed improvement is critical.

Key Whisper parameters configured in Al-Qari:

# From services/transcription.py self.model = WhisperModel( "OdyAsh/faster-whisper-base-ar-quran", device="cuda" if torch.cuda.is_available() else "cpu", compute_type="float16" if device == "cuda" else "int8" ) # Transcription call with Quranic-specific tuning segments, info = model.transcribe( audio_path, language="ar", beam_size=5, # High accuracy for final eval word_timestamps=True, # Required for Tajweed timing vad_filter=True, # Strip silence vad_parameters=dict( min_silence_duration_ms=800, # Allow waqf pauses speech_pad_ms=600 ), temperature=0, # Greedy — deterministic repetition_penalty=1.1 # Suppress tail hallucinations )

📐 Model 2 — Acoustic Tajweed Engine (LibROSA + SciPy)

Rule-based Signal Processing · Phase 1 MVP
Phase 1 (MVP)

The Tajweed checker is not a neural network but a rule-based acoustic analysis engine built on top of LibROSA (audio feature extraction) and SciPy (signal processing). It analyzes the raw audio waveform of each word to verify that Tajweed rules are physically applied correctly.

This approach was chosen for Phase 1 because it is interpretable, fast, and deterministic — the system can explain exactly why a recitation is incorrect (e.g., "Madd too short: 0.3s, expected 0.6s"). A neural Tajweed classifier is planned for Phase 2.

🔊 Supporting Library — PyDub & Audio Pipeline

Audio pre-processing — not an ML model, but a critical pipeline step
Utility

Before audio reaches the Whisper model, it is normalized using PyDub: converted to WAV format, resampled to 16 kHz, and converted to mono. LibROSA then loads the normalized array, which is shared between Whisper (for transcription) and the Tajweed engine (for acoustic analysis) — avoiding duplicate disk reads.

API Endpoints

All REST and WebSocket routes with request/response details

GET / System
Health check — returns API status, version, and available endpoint list.
GET /api/surahs Reference Data
Returns metadata for all 114 Surahs: number, Arabic name, English name, total ayahs, revelation place (Makkah/Madinah).
GET /api/reference/{surah_number}/{ayah_number} Reference Data
Returns full reference data for a specific ayah: Arabic text (Tanzil.net), English and Urdu translations, pre-annotated Tajweed rules (type, position, letter, word, expected duration), and CDN audio URL.
# Example response { "surah": "الفاتحة", "surah_number": 1, "ayah": 1, "arabic_text": "بِسْمِ اللَّهِ الرَّحْمَٰنِ الرَّحِيمِ", "tajweed_rules": [ { "type": "MADD", "position": 2, "madd_type": "madd_asli" } ] }
POST /api/transcribe Transcription
Upload an audio file (WAV/MP3/M4A, max 10 MB) with surah and ayah numbers. Returns AI transcription, confidence score, match percentage vs. reference, and word-by-word comparison.
# multipart/form-data fields audio: <file> # .wav / .mp3 / .m4a surah: 1 # Surah number ayah: 1 # Ayah number # Response { "transcription": "بسم الله الرحمن الرحيم", "confidence": 0.94, "match_percentage": 88.5, "word_comparison": [...] }
POST /api/check-tajweed Tajweed
Full Tajweed analysis: transcribes audio, identifies applied rules, runs acoustic checks for Madd/Qalqalah/Ghunnah, and returns errors with Urdu and English feedback messages.
# Response includes { "overall_accuracy": 85.0, "total_errors": 2, "errors": [ { "rule_type": "MADD", "word": "الرَّحْمَٰنِ", "error_message_en": "Madd too short (0.3s, should be 0.6s)", "error_message_ur": "مد بہت چھوٹی ہے", "measured_value": 0.3, "expected_value": 0.6 } ] }
GET /api/visualize-tajweed/{surah}/{ayah} Visualizer
Returns word-by-word color-coded Tajweed annotations for an ayah — no audio required. Each word includes its rule type, color code, and bilingual label.
MADD — Blue (elongation)
QALQALAH — Orange (echo)
GHUNNAH — Green (nasal)
Neutral — Slate (no rule)
POST /api/evaluate-tajweed Tajweed + Visualizer
Combined endpoint: uploads audio, runs full Tajweed evaluation, and returns annotated words with per-word status (correct / error / missing) and color codes.
WS /ws/recite Real-time
Real-time recitation with live partial transcription.
══ WebSocket Message Protocol ══ # CLIENT → SERVER (text JSON) { "type": "start", "surah": 1, "ayah": 1 } # ayah=0 → full surah mode { "type": "done" } # Signal end of recording # CLIENT → SERVER (binary) <raw PCM/WAV audio bytes, streamed in chunks> ══════════════════════════════════ # SERVER → CLIENT (text JSON) { "type": "ready", "reference_text": "...", "total_rules": 4 } { "type": "partial", "transcription": "بسم الله", "words_heard": 2 } { "type": "result", ...full EvaluatedWordAnnotation list... } { "type": "error", "message": "..." }

Partial transcription fires every ~64 KB (~2s of audio) in single-ayah mode, and every ~480 KB (~15s) in full-surah mode. The server uses a 30-second sliding window to keep Whisper within its context limit.

Data Layer

Quran text, Tajweed rules, and audio data

📖 quran_text.json

Complete Quran text sourced from Tanzil.net — the standard reference for digital Quranic text. Contains all 114 surahs with Arabic text, English and Urdu translations for each ayah, and surah metadata (name, revelation place, total ayahs).

114 Surahs 6,236 Ayahs Tanzil.net Source

📐 tajweed_rules.json

Pre-annotated Tajweed rules for each ayah, keyed by "{surah}:{ayah}". Each entry lists words with their rule type (MADD, QALQALAH, GHUNNAH), word position, Arabic letter, madd type, and expected duration in seconds.

Hand-annotated Position-indexed

🔊 CDN Audio

Audio recitations are served from a public CDN (not stored locally). The reference endpoint returns a CDN URL for each ayah, and the Flutter app streams it directly for playback in the Quran reader. Uploaded user recordings are processed in memory via temporary files that are immediately deleted after analysis.

App Features

All major features of the Al-Qari mobile app

🎤 Recitation Practice

Select any surah/ayah (or full surah mode), record your recitation, and receive AI-powered feedback with word-by-word accuracy scores, Tajweed rule checking, and color-coded word annotations.

📊 Progress Tracking

Every recitation session is saved to Firestore. The progress screen shows accuracy over time with interactive charts (fl_chart), session history, and rolling statistics (total sessions, average accuracy, total minutes).

📚 Quran Reader

Full Quran with authentic Uthmani script from quran_library. Supports word-level audio playback (tap any word to hear it), Tajweed color highlighting, and smooth navigation by surah and page.

📖 Learn Module

Educational content covering Tajweed rules (with bilingual Urdu/English explanations), Qaida (Arabic alphabet basics for beginners), and Qirat (advanced recitation styles).

🔐 Authentication

Email/password signup and login, plus one-tap Google Sign-In via Firebase Auth. User profiles are linked to Firestore for cross-device progress sync.

🔊 Urdu TTS Feedback

Tajweed error messages are read aloud in Urdu using flutter_tts, making feedback accessible to users who are more comfortable with spoken Urdu than reading text.

Core Services

Shared services used across the Flutter app

📈 ProgressService

Saves recitation sessions to Firestore under users/{uid}/progressSessions. Updates rolling stats (total sessions, average accuracy, total duration) in users/{uid}/progress/stats. Handles unauthenticated users gracefully.

🔊 QuranWordAudioService

Plays individual word audio clips from the CDN using the just_audio package. Used in the Quran reader for tap-to-hear word pronunciation.

💾 SessionService

Manages local session state using SharedPreferences. Tracks the current user's reading/recitation session data that should persist across app restarts.

🗣️ UrduTtsService

Wraps flutter_tts to speak Tajweed error messages aloud in Urdu. Configured for Urdu language and appropriate speech rate/pitch for clarity.

⚙️ Model Loading Strategy

AI models are loaded once at server startup using FastAPI's lifespan event and cached in memory as module-level singletons. This avoids reloading the 74M-parameter Whisper model on every request. The first API call may take a few seconds as models download from Hugging Face Hub on a fresh deployment.

# In main.py — models cached at startup async with lifespan(app): _ = get_transcription_service() # Loads + caches Whisper _ = get_tajweed_checker() # Loads + caches TajweedChecker

Glossary

Key terms used in this documentation

TermArabicDefinition
TajweedتجويدThe rules of correct Quranic recitation — pronunciation, rhythm, and articulation.
MaddمدVowel elongation rule. Letters must be held for a specific number of harakaat (counts). Types: Asli (2), Wajib (4-5), Lazim (6).
QalqalahقلقلةEcho/bouncing sound produced on the letters ق ط ب ج د when they carry sukoon. Detected by an intensity spike in the waveform.
GhunnahغنةNasal sound applied to ن and م when followed by specific letters. Detected by nasal frequency energy in the audio spectrum.
WaqfوقفIntentional pause between ayahs in Quranic recitation (0.5–2 seconds). Al-Qari's VAD is configured to tolerate these pauses.
HarakaatحركاتUnit of vowel duration in Tajweed. One harakah ≈ one beat of the finger.
AyahآيةA verse of the Quran.
SurahسورةA chapter of the Quran. There are 114 surahs.
ASR—Automatic Speech Recognition — AI that converts audio to text.
VAD—Voice Activity Detection — filters silence from audio before transcription.
CTranslate2—OpenNMT's optimized inference engine used by Faster-Whisper for INT8/FP16 quantized inference.
Hugging Face Hub—ML model repository where the Quranic Whisper model is hosted and downloaded from.
GetX—Flutter framework for state management, routing, and dependency injection.
